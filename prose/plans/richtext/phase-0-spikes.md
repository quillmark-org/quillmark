# Phase 0 — spikes

Throwaway probes that answer the questions a schema freeze cannot cheaply
revise. Nothing here lands on `main`; each spike exits with a written finding
that unblocks a specific later freeze. Run the three concurrently — they gate
different phases and share no code.

The one rule: **no canonical `RichText` serialization is written until Spike A
reports.** Phase 1 freezes the mark set and its semantics for storage; the
editor spike is what tells us those semantics are the ones a real editor needs.

## Spike A — editor binding (gates the mark-semantics freeze)

**Question.** Which mark semantics must the model encode, and which does it
deliberately leave to the editor? ProseMirror, Lexical, and Quill disagree
exactly where the canonical serialization commits: mark identity across
split/join, adjacent same-type merge, and edge-expand (does typing at a bold
boundary extend the bold?).

**Do.** Bind at least two surfaces against a throwaway corpus: a rich prose
editor (ProseMirror or Lexical) and a plaintext/markdown surface
(CodeMirror-style). Drive the edits the model must survive:

- Split and join a paragraph mid-mark; observe what each editor does to mark
  ranges and to a zero-width identity mark.
- Overlap two same-type and two different-type marks; confirm the union /
  free-overlap normalization holds under editor transactions.
- Type at both edges of a mark; record the expand behavior each editor assumes.
- Run the MCP full-document-rewrite path (cold parse + corpus diff) with an
  annotation present, including a **paragraph reorder**, and measure annotation
  survival — this is where the move weak spot bites.

**Exit.** A written mark-semantics contract: the frozen set, the normalization
rules, and an explicit list of what the model does *not* encode (edge-expand and
adjacent-merge as editor/CRDT concerns). Names the differ's move-detection
policy or accepts the drop and states it. **Unblocks phase 1's canonical
serialization and phase 3's editor binding.**

## Spike B — source-map inversion + navigation (gates the phase-2 emit design)

**Question.** Does glyph-offset inversion actually yield a usable point↔corpus
map through `escape_markup`, at cluster granularity, on real typeset output?

**Do.** On an existing fixture render, without building the full model:

- Emit one content field as a `#let` block recording, per text run, a
  `(corpus range ↔ generated range)` pair.
- Use `glyph.span.0` **and** `glyph.span.1` (the intra-node offset unused today
  at `span_scan.rs:197`) to resolve a page hit to a generated byte, invert the
  run map, and land a corpus position. Round-trip it back to a page rect.
- Probe the boundaries: ligatures and combining marks (cluster, not char),
  escaped characters (one corpus char → several generated bytes), and ink with
  no corpus origin (list markers, package numbering) — confirm these are
  cleanly *nav-ignored*, not mis-mapped.

**Exit.** A finding on achievable granularity (expected: cluster-exact, and the
plan says "character-exact" nowhere it means cluster) and on whether the run map
is invertible by recomputation with no per-character tables. Flags any escape
construct that breaks run-alignment. **Unblocks phase 2's emit + source-map
design.** Does *not* attempt the `regions()` run-machine rework (grounding §3.2)
— that is phase-2 work, scoped separately.

## Spike C — seam encoding + determinism (confirms Option A, gates phase 2 storage)

**Question.** Does canonical RichText-JSON on the data seam serialize
byte-deterministically, align with the storage serialization, and lower
trivially to both backends?

**Do.**

- Prototype a RichText value and its canonical JSON form; hash it twice, confirm
  byte-identity, and confirm the seam JSON and the storage serialization are the
  same canonical bytes (one encoding, not two to keep aligned).
- Lower it two ways: `typst` codegen (structure → markup, throwaway) and the
  `pdfform` `RichText → .text` path (island slots dropped). Confirm `pdfform`
  needs no fixture change (`sample_form` binds only scalars).
- Author one island, mint its ID, and confirm where mint-nondeterminism enters
  the hash — documenting the boundary the content-hash contract must tolerate
  from phase 4 on.

**Exit.** Confirmation (or a red flag) on Option A's determinism and dual-lower
story, and a one-paragraph note on the island-mint hash boundary. **Unblocks
phase 2's seam + storage cutover.**

## Gate

Phase 1 opens when Spike A reports. Phase 2's emit design opens when Spike B
reports; its storage cutover when Spike C reports. A spike that surfaces a
red flag reshapes the phase it gates before that phase starts — that is the
point of running them first.
